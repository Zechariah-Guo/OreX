Welcome to your AT3 Major Project repository.  
This project mirrors real‑world software development: planning, modelling, building, testing, documenting, and presenting a complete software solution.

This README explains:

- What this project is  
- How the repository is structured  
- What you must produce  
- Where everything goes

# 🚀 Project Summary

In this project, you will:

1. **Design** a software solution using industry‑standard modelling tools  
2. **Build** the solution using clean, maintainable code  
3. **Test** your solution thoroughly  
4. **Document** your work professionally  
5. **Present** your solution  
6. **Submit** everything through GitHub Classroom and Google Classroom  

Your work must demonstrate:

- Problem‑solving  
- System design  
- Programming skills  
- Testing and evaluation  
- Communication and documentation

# 📁 Repository Structure

Your repository contains the following folders:

```
src/                → Your source code
docs/               → Documentation
docs/modelling/     → Modelling templates (you complete these)
deliverables/       → Final packaged submission (all components)
.github/            → Automated checks + issue templates
START_HERE.md       → Your onboarding guide
ChecklistTemplate.md → Your full pre‑submission checklist
README.md           → This file
```

# 🧩 Required Modelling Tools

You must complete **all** of the following modelling artefacts:

- Class Diagram  
- Use Case Diagram  
- System Flow Diagram  
- Data Dictionary  
- User Stories  
- Decision Tree  
- Context Diagram (Level 0 DFD)  
- Level 1 DFD  
- Structure Chart  
- Storyboard  

Templates are in:

```
docs/modelling/
```

Your completed versions go in:

```
deliverables/modelling/
```

All diagrams must:

- Render correctly on GitHub  
- Use correct notation  
- Match your implementation  

---

# 📚 Required Documentation

All documentation must be placed in:

```
deliverables/documentation/
```

You must include:

- Project Proposal  
- User Stories (final)  
- Requirements summary  
- All modelling tools  
- Testing Log  
- Reflection / Evaluation  
- Bibliography / References  
- Final compiled documentation PDF  

Your PDF must include **all** documentation and modelling.

---

# 💻 Required Implementation Artefacts

All implementation artefacts must be placed in:

```
deliverables/implementation/
```

You must include:

- Fully working program  
- All source code in `src/`  
- README with run instructions  
- Screenshots of the running system  
- Evidence of testing  
- Version control history (regular commits)  

Your program must run without errors and match your modelling.

---

# 🎤 Required Presentation Artefacts

All presentation materials must be placed in:

deliverables/presentation/

You must include:
