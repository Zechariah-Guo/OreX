# 📚 Data Dictionary — OreX

A **Data Dictionary** defining every data item used in the OreX system,
including types, descriptions, examples, and validation rules.

---

## Users Table

| Data Item | Type | Description | Example Value | Validation Rules |
|-----------|------|-------------|---------------|------------------|
| id | INTEGER | Unique user identifier (auto-incremented primary key) | 1 | Auto-generated; positive integer |
| username | TEXT | User's display name and login identifier | "DiamondTrader" | 3–20 characters; alphanumeric and underscores only; must be unique |
| password_hash | TEXT | Werkzeug PBKDF2 hash of the user's password | "pbkdf2:sha256:..." | Generated by `generate_password_hash`; never stored as plaintext |
| balance | REAL | Current virtual cash balance in dollars | 10000.00 | Cannot be negative; defaults to 10000; updated on buy/sell |
| created_at | TEXT | ISO 8601 timestamp of account creation | "2026-05-20T14:30:00" | Set automatically on registration; not user-editable |
| last_login | TEXT | ISO 8601 timestamp of most recent login | "2026-06-15T09:12:45" | Nullable; updated on each successful login |

## Ores Table

| Data Item | Type | Description | Example Value | Validation Rules |
|-----------|------|-------------|---------------|------------------|
| id | INTEGER | Unique ore identifier (primary key) | 4 | Auto-generated; positive integer |
| name | TEXT | Display name of the ore | "Gold" | Must be unique; not empty |
| description | TEXT | Flavour text describing the ore | "Mid-tier, balanced. Prized for its lustre." | Optional; nullable |
| icon_filename | TEXT | Filename of the ore's icon image | "Gold_Ingot.webp" | Optional; must match a file in static/images/ores/ |
| current_price | REAL | Live market price updated each tick | 52.34 | Must be between price_floor and price_ceiling |
| base_price | REAL | Anchor price for gravity/mean reversion calculation | 50.00 | Positive; set at seed time; not changed at runtime |
| price_floor | REAL | Minimum allowed price | 10.00 | Positive; less than base_price |
| price_ceiling | REAL | Maximum allowed price | 200.00 | Positive; greater than base_price |
| volatility | REAL | Scaling factor for price movement magnitude | 0.9 | Range 0.0–2.0; higher means more volatile |
| price_change_range | TEXT | JSON array [min%, max%] for tick calculation | "[2, 7]" | Valid JSON; two positive numbers; min < max |
| base_probabilities | TEXT | JSON array [rise%, hold%, fall%] | "[30, 45, 25]" | Valid JSON; three numbers summing to 100 |
| trend_log | TEXT | JSON array of last 5 price movements | '["rise","hold","fall","rise","hold"]' | Valid JSON; exactly 5 entries; each "rise", "hold", or "fall" |

## Holdings Table

| Data Item | Type | Description | Example Value | Validation Rules |
|-----------|------|-------------|---------------|------------------|
| id | INTEGER | Unique holding identifier (primary key) | 12 | Auto-generated; positive integer |
| user_id | INTEGER | Foreign key referencing users.id | 1 | Must exist in users table |
| ore_id | INTEGER | Foreign key referencing ores.id | 4 | Must exist in ores table |
| quantity | INTEGER | Number of ore units the user holds | 25 | Positive integer; row deleted when reaching 0 |
| avg_purchase_price | REAL | Weighted average cost per unit | 48.50 | Positive; recalculated on each additional purchase |

## Transactions Table

| Data Item | Type | Description | Example Value | Validation Rules |
|-----------|------|-------------|---------------|------------------|
| id | INTEGER | Unique transaction identifier (primary key) | 156 | Auto-generated; positive integer |
| user_id | INTEGER | Foreign key referencing users.id | 1 | Must exist in users table |
| ore_id | INTEGER | Foreign key referencing ores.id | 4 | Must exist in ores table |
| type | TEXT | Type of trade executed | "buy" | Must be "buy" or "sell" |
| quantity | INTEGER | Number of units traded | 10 | Positive integer |
| price_at_trade | REAL | Unit price at the time of trade | 52.34 | Positive; captured from current_price at execution |
| total_value | REAL | Total cost or proceeds (quantity × price) | 523.40 | Positive; equals quantity × price_at_trade |
| archived | INTEGER | Whether the transaction has been archived by account reset | 0 | 0 (active) or 1 (archived); defaults to 0 |
| created_at | TEXT | ISO 8601 timestamp of trade execution | "2026-06-15T10:05:30" | Set automatically; not user-editable |

## Price History Table

| Data Item | Type | Description | Example Value | Validation Rules |
|-----------|------|-------------|---------------|------------------|
| id | INTEGER | Unique record identifier (primary key) | 4521 | Auto-generated; positive integer |
| ore_id | INTEGER | Foreign key referencing ores.id | 4 | Must exist in ores table |
| price | REAL | Recorded price at this tick | 52.34 | Positive; matches ores.current_price after the tick |
| movement | TEXT | Direction of price change at this tick | "rise" | Must be "rise", "hold", or "fall" |
| created_at | TEXT | ISO 8601 timestamp of the tick | "2026-06-15T10:05:20" | Set automatically each tick |

## Application Configuration

| Data Item | Type | Description | Example Value | Validation Rules |
|-----------|------|-------------|---------------|------------------|
| SECRET_KEY | TEXT | Flask session encryption secret | "orex-local-dev-secret..." | Non-empty string; should be changed for production |
| DATABASE_PATH | TEXT | File path to the SQLite database | "data/orex.db" | Valid file path; directory created if missing |
| DEFAULT_BALANCE | INTEGER | Starting balance for new user accounts | 10000 | Positive integer |
| TICK_INTERVAL | INTEGER | Seconds between market price updates | 20 | Positive integer; minimum 5 |
| RATE_LIMIT_WINDOW | INTEGER | Rate limit time window in seconds | 900 | Positive integer (15 minutes = 900) |
| RATE_LIMIT_MAX | INTEGER | Maximum login attempts per window | 5 | Positive integer |

## Input Validation Rules

| Input Field | Source | Validation | Error Message |
|-------------|--------|-----------|---------------|
| Username (registration) | Form POST | 3–20 chars; alphanumeric + underscore; unique | "Username must be at least 3 characters." / "Username can only contain letters, numbers, and underscores." |
| Password (registration) | Form POST | Minimum 8 characters; must contain at least one uppercase letter, one lowercase letter, one number, and one symbol | "Password must be at least 8 characters." / "Password must contain at least one number." / etc. |
| Confirm Password | Form POST | Must match password field | "Passwords do not match." |
| Trade Quantity | Form POST | Positive integer | "Quantity is required." / "Quantity must be a whole number." / "Quantity must be greater than zero." |
| Reset Confirmation | Form POST | Must exactly match current username | "Username confirmation does not match." |
